Terminal Also: Console, Command Line
Shell / CLI

A text-based window on your computer where you type commands directly. Instead of clicking buttons, you type instructions. The Terminal is the window itself; what runs inside it is a shell.

Analogy: The Terminal is the room. The shell is the person inside who understands your instructions.
Shell bash, zsh, fish, sh
Shell / CLI

The program that runs inside a Terminal and interprets what you type. When you open Terminal on a Mac, you're almost certainly running a shell called zsh (Z Shell) or bash. It reads your commands and tells the operating system what to do.

See also: Terminal, bash, zsh, PowerShell

Analogy: The translator between you and the computer's core. You speak; the shell passes it on in a language the OS understands.
bash Bourne Again SHell
Shell / CLI

The most widely used shell in the world. It's been the default on Linux and older Macs for decades. When someone shares a "bash script," it's a text file full of commands that bash will run in sequence, like a recipe.

Analogy: The universal language of command lines. If Terminal is the room, bash is the dialect almost everyone speaks.
zsh Z Shell
Shell / CLI

A modern shell that replaced bash as the Mac default in 2019. It's more powerful with better autocomplete and plugin support. If you've opened Terminal on a modern Mac, you've been using zsh without knowing it.

Analogy: bash's more capable younger sibling. Same idea, better features.
PowerShell Windows shell
Shell / CLI

Microsoft's command-line shell for Windows. Much more powerful than the older Command Prompt (cmd.exe). It can also run on Mac and Linux. Commands in PowerShell are called "cmdlets" and follow a Verb-Noun pattern like Get-Files or Set-Location.

Windows equivalent of: bash / zsh on Mac/Linux

Analogy: Microsoft's answer to bash. Same category of tool, built for Windows first.
cmd / Command Prompt cmd.exe
Shell / CLI

The original Windows command line, dating back to the DOS era. Still available on all Windows machines. Very limited compared to PowerShell. You'll occasionally see instructions reference it, but PowerShell is almost always better.

Analogy: The old DOS era tool still kept alive. Functional, but outdated.
SSH Secure Shell
Network / Protocol

A way to securely connect to another computer over the internet and control it via the command line, as if you were sitting in front of it. Used constantly by developers to access remote servers. The connection is encrypted so no one can spy on what you're doing.

Related: Terminal (you SSH via it), servers, GitHub authentication

Analogy: A secure, private telephone line between your laptop and a computer sitting in a data center on the other side of the world.
npm Node Package Manager
Tool / App

A tool that comes bundled with Node.js. It lets you install, share and manage code packages (libraries written by other developers) that your project depends on. When someone says "run npm install," they mean: download all the code this project needs to work.

Requires: Node.js | Alternative: yarn, pnpm

Analogy: The App Store for JavaScript code. Instead of building everything from scratch, you install packages someone else already wrote.
Node.js JavaScript runtime
Language / Runtime

JavaScript was originally only for browsers. Node.js lets you run JavaScript on your computer or a server, outside of any browser. This opened up JavaScript to backend development and tooling. Many dev tools (including npm, Claude Code, and Vite) are built on Node.js.

Enables: npm, Claude Code, most JS dev tools

Analogy: Taking JavaScript out of the browser and letting it run anywhere, like a car engine removed and put in a boat.
Git Version Control System
Tool / App

A system that tracks every change you make to your code over time. You can go back to any previous version, work on experimental features in separate "branches," and merge changes together. Git runs locally on your machine. GitHub is a website that hosts your Git history online.

Not the same as: GitHub (GitHub hosts Git repositories online)

Analogy: An infinite undo history for your entire project, with the ability to create parallel timelines and merge them back.
GitHub Code hosting platform
Tool / App

A website that stores your Git repositories online. Think of it as cloud storage specifically for code, with collaboration features built in. You push your code from your machine to GitHub. Others can see it, contribute, and you can deploy from it.

Powered by: Git | Competitors: GitLab, Bitbucket

Analogy: Google Drive, but designed specifically for code, with version history baked in.
CLI Command Line Interface
Concept

Any tool or program you control entirely by typing text commands, rather than clicking buttons. The opposite is a GUI (Graphical User Interface). Developers use CLIs because they're fast, scriptable, and work on any machine. When docs say "run this in your CLI," they mean open Terminal and type it.

Analogy: Giving instructions by typing a precise note versus navigating menus. More effort to learn, but much faster once you know it.
GUI Graphical User Interface
Concept

Any interface you interact with visually: buttons, menus, windows. Most apps people use daily are GUIs. The opposite of a CLI. When someone builds a "frontend," they're building a GUI. VS Code has a GUI, but also accepts CLI commands.

Analogy: Everything you normally click on a computer. Most of the digital world is a GUI.
API Application Programming Interface
Concept

A defined way for two pieces of software to talk to each other. When your app asks Supabase for data, it uses Supabase's API. When you use Anthropic's Claude in code, you call the Claude API. APIs have rules: send a request in this format, and you'll get a response in that format.

Analogy: A waiter in a restaurant. You don't go into the kitchen yourself. You tell the waiter (API) what you want, and it comes back with the result.
HTTP / HTTPS HyperText Transfer Protocol (Secure)
Network / Protocol

The rules that govern how data travels between your browser and a website. HTTP is the base version; HTTPS adds encryption (the S stands for Secure). Every website URL starts with one of these. HTTPS is the standard now; browsers flag plain HTTP as unsafe.

Analogy: The postal rules that define how a letter gets from sender to receiver. HTTPS adds a sealed envelope so no one can read it in transit.
localhost 127.0.0.1
Concept

A special address that always refers to your own computer. When you run a web server locally for development, you visit localhost:3000 (or similar) in your browser to see it. The number after the colon is the port. Nothing on localhost is visible to anyone else online.

Related: dev server, port, IP address

Analogy: A private preview only you can see, running on your own machine before anything goes live on the internet.
Port Network port number
Network / Protocol

A numbered "channel" on a computer that a specific program listens on. Your computer has thousands of ports. Port 80 is standard HTTP, 443 is HTTPS, 3000 and 8080 are common development ports. When you see localhost:3000, the 3000 is the port.

Analogy: Your home address is the computer. The port is the specific apartment number within it. Multiple programs live at the same address on different ports.
DNS Domain Name System
Network / Protocol

The internet's phone book. When you type "google.com" in a browser, DNS converts that human-readable name into the actual numeric IP address that computers use to find the server. Without DNS, you'd need to remember IP addresses to visit any website.

Analogy: Saving contacts by name on your phone. You say "call Mom," the phone looks up her number. DNS does the same for websites.
IP Address Internet Protocol Address
Network / Protocol

A unique numerical address assigned to every device on a network. Like a postal address for computers. IPv4 addresses look like 192.168.1.1. IPv6 is a newer format to handle more devices. Your router has a public IP; devices on your home network have local IPs.

Analogy: Your home's street address, but for computers on a network.
Server Web server / remote machine
Concept

A computer (or program) that waits for requests and responds to them. A web server receives HTTP requests and sends back web pages. A database server stores and retrieves data. "Server" can mean the physical machine in a data center, or just the software program responding to requests.

Analogy: A restaurant that's always open. You place an order (send a request), it prepares and returns what you asked for (sends a response).
Database SQL, Postgres, Supabase
Concept

An organized system for storing, retrieving and managing data. Relational databases (like PostgreSQL, which powers Supabase) store data in tables with rows and columns, like a very powerful spreadsheet. You query them using SQL to get the data you need.

Analogy: An enormous, perfectly organized filing system that can answer questions like "give me every user who signed up last month" instantly.
SQL Structured Query Language
Language / Runtime

The language used to talk to relational databases. You write SQL to ask questions ("SELECT all users"), add data ("INSERT a new record"), or change data ("UPDATE this row"). It reads almost like English. Most backend developers need to know at least basic SQL.

Analogy: The language for asking questions to a database. "Give me all rows where the city is Cape Town" is practically what the SQL looks like.
Frontend / Backend Client-side / Server-side
Concept

Frontend is everything the user sees and interacts with: the interface in their browser. Backend is everything happening on the server: data storage, logic, authentication. Full-stack means working on both.

Analogy: A restaurant again. Frontend is the dining room and menu. Backend is the kitchen, inventory, and accounting.
Deploy Deployment / Ship / Go live
Concept

The act of publishing your code so real users can access it on the internet. Before deployment, your app lives only on your machine. After deployment, it's live at a URL. Netlify handles this automatically when you push code to GitHub.

Your deploy platform: Netlify

Analogy: Printing a document and handing it out. Development is writing the draft; deployment is publishing it to the world.
Repository Repo
Concept

A folder that Git tracks. Every project typically lives in its own repository. It contains all your code files plus a hidden .git folder where Git stores the entire history of changes. On GitHub, a repository is a project's home with all its history, branches, and settings.

Analogy: A binder with every draft of a document you've ever saved, labeled by date and description, never throwing anything away.
Branch Git branch
Concept

A separate, parallel version of your code within the same repository. The main branch is typically the stable, live version. You create a new branch to work on a feature or experiment without affecting main. When the work is ready, you merge the branch back in.

Analogy: A photocopy of a document you're allowed to scribble on. The original stays safe until you decide to apply your changes.
Commit Git commit
Concept

A saved snapshot of your code at a specific point in time. Every commit has a message describing what changed. Git history is just a sequence of commits. Think of commits as manual save points that stack up into a timeline you can always scroll back through.

Analogy: A checkpoint in a game. You can always load a previous save if something goes wrong.
Push / Pull Git push, git pull
Concept

Push: sending your local commits up to GitHub (or another remote). Pull: downloading commits from GitHub to your local machine. These two commands keep your local code and the online repository in sync. You push your changes; your collaborators pull them down.

Analogy: Push is uploading a file to Google Drive. Pull is downloading changes someone else made to a shared document.
.env / Environment Variables Secrets and config
Concept

Variables stored outside your code that hold sensitive information like API keys, passwords, and database URLs. A .env file keeps these off your actual codebase so they don't accidentally get committed to GitHub for the public to see.

Analogy: A locked safe next to your open notebook. The notebook holds instructions; the safe holds the keys and passwords it references.
Netlify Frontend hosting / deployment
Tool / App

A platform that hosts your frontend (HTML/CSS/JS) and deploys it automatically when you push to GitHub. Free tier is generous. It handles custom domains, HTTPS certificates, and serverless functions.

Stack: Netlify (frontend) + Supabase (backend/database)

Analogy: A landlord who automatically repaints your shop window every time you send them an update, at no charge for basic use.
Supabase Backend as a Service
Tool / App

An open-source platform that gives you a PostgreSQL database, user authentication, real-time subscriptions, and file storage, all without managing a server. You get a dashboard to manage your data and an API to connect your frontend.

Analogy: A fully staffed back-office. It handles your filing, security, and records while you focus on the customer-facing side.
VS Code Visual Studio Code — code editor
Tool / App

Microsoft's free code editor, arguably the most popular in the world. It's where most developers write their code. It has syntax highlighting, autocomplete, Git integration built in, and thousands of extensions.

Analogy: Microsoft Word, but built specifically for writing code, with tools that understand programming languages.
Claude Code Anthropic's agentic coding CLI
Tool / App

Anthropic's command-line tool that lets Claude (the AI) read, write, and edit files in your project directly from your Terminal. It understands your codebase and can make changes across multiple files. It runs on Node.js.

Requires: Node.js, Terminal access

Analogy: Hiring a developer who can see your entire codebase and make changes, but you direct them via conversation in your Terminal.
MCP Model Context Protocol
Concept

An open standard created by Anthropic that lets AI models (like Claude) connect to external tools and services in a standardized way. MCP creates a common language so AI tools can plug into Google Drive, Notion, GitHub, and more consistently.

Analogy: A universal power adapter. Instead of a different adapter for every country (tool), MCP is one standard plug that works everywhere.
Package / Dependency Library, module
Concept

Reusable code someone else wrote that your project uses. Instead of writing every feature yourself, you install packages. A dependency is any package your project depends on to function. The package.json file lists all dependencies so anyone can install them with npm install.

Managed by: npm, yarn, pip (Python), etc.

Analogy: Pre-made ingredients. You don't grind your own coffee beans when you can buy a bag. Packages are those bags of pre-made capability.
Python Programming language
Language / Runtime

One of the world's most popular programming languages. Known for its clean, readable syntax. Widely used in data science, AI/ML, automation, and backend web development.

Analogy: The plain English of programming languages. Closer to human language than most, which makes it beginner-friendly but also powerful enough for cutting-edge AI work.
JavaScript JS — the language of the web
Language / Runtime

The only programming language that runs natively in every web browser. It makes websites interactive: form validation, animations, fetching data without reloading. Node.js lets it run outside browsers too.

Analogy: HTML is the skeleton of a webpage, CSS is the styling, and JavaScript is the muscles that make it move and respond.
HTML / CSS Structure and style
Language / Runtime

HTML (HyperText Markup Language) defines the structure and content of a web page: headings, paragraphs, buttons, images. CSS (Cascading Style Sheets) controls how it looks: colors, fonts, layout, spacing. They are not programming languages; they are markup and styling languages.

Analogy: HTML is the blueprint of a building (what rooms exist). CSS is the interior design (how everything looks and is arranged).
Open Source OSS — publicly available code
Concept

Software whose source code is publicly available for anyone to read, use, modify, and contribute to. Git, Linux, Node.js, Python, and VS Code are all open source. The opposite is proprietary software, where the code is private.

Analogy: A recipe book anyone can read, copy, modify, and contribute to. Versus a secret recipe locked in a vault.

All terms accurate as of 2025 — dev terminology evolves, concepts stay the same.

React JavaScript UI library
Framework

A JavaScript library by Meta for building user interfaces. React breaks UIs into reusable components — each a self-contained piece of code that manages its own state and renders HTML. It's the most widely used frontend tool and the foundation of Next.js.

See also: Next.js, JavaScript, Node.js

Analogy: LEGO bricks. Each component is a brick with a defined shape. You assemble them into any structure you want.
Vue.js Progressive JavaScript framework
Framework

A progressive JavaScript framework for building web UIs. Simpler to learn than React, still very capable. "Progressive" means you can adopt it incrementally — drop it into an existing page or use it to build a full app from scratch.

See also: React, JavaScript

Analogy: A more opinionated LEGO set. Less total flexibility than React, but the instructions are clearer and the pieces fit together faster.
Next.js React framework
Framework

A framework built on top of React that adds server-side rendering, file-based routing, and performance optimisations. If React is the engine, Next.js is the whole car. It handles the parts React leaves to you: URL routing, data loading, image optimisation.

Built on: React | Hosting: Vercel, Netlify

Analogy: React with everything pre-assembled — engine, chassis, GPS, and fuel-efficiency baked in.
Express.js Node.js web framework
Framework

The most popular Node.js framework for building web servers and APIs. Minimal and flexible — it gives you routing and middleware, and you add what you need. Most Node.js backend tutorials use Express as their starting point.

Requires: Node.js | See also: API, Server

Analogy: Bare metal for web servers in JavaScript. You get the wiring; you decide what plugs in where.
Django Python web framework
Framework

A batteries-included Python web framework. It ships with an admin interface, database ORM, user authentication, and more out of the box. Its philosophy: don't repeat yourself. Popular for building data-heavy web applications quickly.

Language: Python | See also: FastAPI, SQL

Analogy: A fully furnished apartment. Move in and start working. No assembling furniture required.
FastAPI Python API framework
Framework

A modern Python framework built specifically for APIs. Much faster at runtime than Django for pure API work. Automatically generates interactive documentation from your code. Widely used in AI and data science backends.

Language: Python | See also: Django, API

Analogy: A race car version of a Python API server. Stripped down and purpose-built for speed.
Tailwind CSS Utility-first CSS framework
Framework

A CSS framework that provides utility classes instead of pre-built components. Instead of writing custom CSS, you apply classes like flex, text-xl, and bg-black directly in your HTML. Controversial at first glance, extremely popular in practice.

See also: HTML / CSS, Bootstrap

Analogy: A massive set of labelled materials — every brick, plank, and fitting pre-cut to size. No pre-built rooms, but you can build anything fast.
Bootstrap CSS component framework
Framework

The original CSS framework. Provides pre-styled components: buttons, navbars, modals, grids. Widely used, easy to get started with. Sometimes criticised for making sites look similar. Tailwind has replaced it in many newer projects.

See also: HTML / CSS, Tailwind CSS

Analogy: A furniture kit from a catalogue. Pick what you need, snap it in. Faster than building from scratch, less unique.

All terms accurate as of 2025 — dev terminology evolves, concepts stay the same.

TypeScript Typed JavaScript superset
Language / Runtime

A superset of JavaScript created by Microsoft. It adds static typing — you declare what type a variable should hold, and TypeScript catches errors before your code runs. All valid JavaScript is valid TypeScript. It compiles down to plain JavaScript to run in the browser or Node.js.

See also: JavaScript, Node.js

Analogy: JavaScript with spell-check. The extra rules slow you down initially but prevent entire categories of bugs before they ever reach production.
Go Golang — compiled systems language
Language / Runtime

A compiled, statically typed language designed at Google for simplicity and performance. Known for excellent handling of concurrent tasks. Popular for building APIs, CLIs, and infrastructure tools. Compiles to a single binary with no runtime dependencies — just copy the file and run it.

See also: Python, Rust

Analogy: A purpose-built delivery van. Not the flashiest vehicle, but reliable, fast, and built to carry heavy loads efficiently without drama.
Rust Systems programming language
Language / Runtime

A systems language focused on memory safety and performance. No garbage collector — memory is managed at compile time via an ownership system. Steep learning curve, but produces extremely fast and safe code. Popular for CLIs, WebAssembly, and systems-level work.

See also: Go, C / C++

Analogy: A manual car with extra safety interlocks. Harder to drive, but you understand exactly what's happening under the hood and nothing can silently go wrong.
Ruby Dynamic, expressive language
Language / Runtime

A dynamic language designed for programmer happiness. Known for Ruby on Rails, the framework that defined modern web development conventions. Less dominant than it once was, but the ideas it introduced — convention over configuration — shaped every framework that followed.

Famous for: Ruby on Rails | See also: Python

Analogy: The language built around the developer's joy. Syntax reads close to natural English, and it shaped how the whole industry thought about web frameworks.
Swift Apple's programming language
Language / Runtime

Apple's language for building iOS and macOS apps. It replaced Objective-C as the standard for Apple development. Fast, safe, and modern in syntax. If you want to build a native iPhone or Mac app, Swift is where you start.

Platform: iOS, macOS | See also: Kotlin (Android)

Analogy: The language Apple built specifically for its own world. Fluent in iOS and macOS; less relevant outside Apple's ecosystem.
Kotlin Modern JVM language
Language / Runtime

Google's preferred language for Android app development. It runs on the Java Virtual Machine (JVM) and is fully interoperable with Java. More concise and safer than Java, with modern language features. Also used for backend development via frameworks like Ktor.

Platform: Android, JVM | See also: Swift (iOS)

Analogy: Java's more modern, friendlier replacement. Same runtime underneath, far less boilerplate on top.

Axios HTTP request library
Library

A JavaScript library for making HTTP requests — calling APIs — from the browser or Node.js. More ergonomic than the built-in fetch API. Handles JSON parsing, request timeouts, and error formatting in a cleaner, more predictable way.

Language: JavaScript | See also: API, HTTP / HTTPS

Analogy: A polished postal service. You hand it a letter (request); it handles the delivery and brings back the reply already opened and readable.
Lodash JavaScript utility library
Library

A JavaScript utility library with hundreds of helper functions for working with arrays, objects, and strings. Reduces repetitive code. Functions like groupBy, flatten, and debounce save writing the same logic over and over across every project.

Language: JavaScript | Installed via: npm

Analogy: A kitchen full of specialty tools. You could do everything with a knife, but a mandolin, a zester, and a peeler make the job faster and cleaner.
jQuery DOM manipulation library
Library

Once the dominant JavaScript library. Made it easy to manipulate HTML elements, handle events, and make requests across different browsers. Modern JavaScript has absorbed most of what jQuery offered. Legacy codebases still use it widely — you'll encounter it.

Language: JavaScript | See also: JavaScript, DOM

Analogy: The utility belt web developers wore for a decade. Mostly replaced by native browser tools, but still running under the hood of a huge amount of existing web software.
D3.js Data visualisation library
Library

A JavaScript library for creating data visualisations in the browser. Extremely powerful and flexible — you define exactly how data maps to visual shapes and positions. Used for custom charts, maps, and interactive graphics where off-the-shelf chart tools aren't enough.

Language: JavaScript | See also: Chart.js

Analogy: A professional illustrator's toolkit. You can draw anything imaginable, but you draw it yourself, line by line.
Chart.js Simple charting library
Library

A JavaScript library for drawing common charts and graphs. Much simpler than D3.js — pick a chart type, pass in data, get a visual. Covers most everyday needs: bar, line, pie, doughnut, radar. Good default aesthetics with minimal setup.

Language: JavaScript | See also: D3.js

Analogy: A chart builder where you fill in the spreadsheet and choose a style. D3.js is the artist; Chart.js is the template.
Pandas Python data analysis library
Library

Python's go-to library for data analysis and manipulation. Provides DataFrames — table-like structures that let you load, clean, filter, and transform data with minimal code. Essential in data science, analytics, and any Python work that involves spreadsheets or databases.

Language: Python | See also: NumPy, SQL

Analogy: Excel, but scriptable and infinitely more powerful. A table that can reshape, filter, and calculate itself on command.
NumPy Python numerical computing library
Library

Python's numerical computing library. Provides multi-dimensional arrays and mathematical functions that run far faster than native Python. The foundation of Python's scientific and AI computing ecosystem — Pandas, TensorFlow, and most ML libraries are built on top of it.

Language: Python | See also: Pandas

Analogy: The engine underneath most of Python's data and AI work. Most people never call it directly, but it's running everything.

All terms accurate as of 2025 — dev terminology evolves, concepts stay the same.
